# Open Brain — Supabase Setup Instructions

> **What this is:** A shared AI memory system. Captures thoughts from Slack (and any MCP client), stores them as vector embeddings in Supabase pgvector, and exposes semantic search via an MCP server that any AI tool can connect to.
>
> **Prerequisites:** Supabase project, OpenRouter API key, Slack workspace (optional for capture)

---

## Phase 1: Database Setup

### 1.1 Enable pgvector extension

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 1.2 Create the thoughts table

```sql
CREATE TABLE thoughts (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  content TEXT NOT NULL,
  embedding vector(1536),
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX thoughts_embedding_idx ON thoughts
  USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

CREATE INDEX thoughts_created_at_idx ON thoughts (created_at DESC);
```

### 1.3 Create the vector search function

```sql
CREATE OR REPLACE FUNCTION match_thoughts(
  query_embedding vector(1536),
  match_threshold float DEFAULT 0.7,
  match_count int DEFAULT 10
) RETURNS TABLE (
  id bigint,
  content text,
  metadata jsonb,
  similarity float
) LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY
  SELECT
    t.id,
    t.content,
    t.metadata,
    1 - (t.embedding <=> query_embedding) AS similarity
  FROM thoughts t
  WHERE 1 - (t.embedding <=> query_embedding) > match_threshold
  ORDER BY t.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
```

### 1.4 Enable Row Level Security

```sql
ALTER TABLE thoughts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Service role only"
  ON thoughts FOR ALL
  USING (auth.role() = 'service_role');
```

## Phase 2: Capture Pipeline (Edge Function)

### 2.1 Set Supabase secrets

```bash
supabase secrets set OPENROUTER_API_KEY=sk-or-v1-your-key-here
supabase secrets set SLACK_BOT_TOKEN=xoxb-your-bot-token
supabase secrets set SLACK_CAPTURE_CHANNEL=C0123456789
```

### 2.2 Create and deploy the ingest function

```bash
supabase functions new ingest-thought
```

The edge function should:
1. Receive Slack webhook events (message posts to the capture channel)
2. Filter: only process `message` events (not `message_changed`, bot messages, etc.)
3. In parallel:
   - Generate embedding via OpenRouter (`text-embedding-3-small`, 1536 dims)
   - Extract metadata via OpenRouter (`gpt-4o-mini`) — type, tags, people, action items, priority
4. Insert row into `thoughts` table with content, embedding, and metadata JSONB
5. Return 200 within 3 seconds (Slack's webhook timeout)

**Embedding API call (OpenRouter):**
```
POST https://openrouter.ai/api/v1/embeddings
Headers: Authorization: Bearer $OPENROUTER_API_KEY
Body: { "model": "openai/text-embedding-3-small", "input": "thought text here" }
```

**Metadata extraction prompt (gpt-4o-mini via OpenRouter):**
```
POST https://openrouter.ai/api/v1/chat/completions
Model: openai/gpt-4o-mini
System: Extract structured metadata from this thought. Return JSON:
{
  "type": "person_note|action_item|insight|observation|decision|question",
  "tags": ["category1", "category2"],
  "people": ["name1"],
  "action_items": ["task"],
  "priority": "high|medium|low"
}
```

### 2.3 Deploy with no JWT verification (webhooks are unauthenticated)

```bash
supabase functions deploy ingest-thought --no-verify-jwt
```

### 2.4 Configure Slack app

1. Create a Slack app at https://api.slack.com/apps
2. **OAuth Scopes:** `channels:history`, `groups:history`, `chat:write`
3. **Event Subscriptions → Request URL:** `https://[PROJECT_REF].supabase.co/functions/v1/ingest-thought`
4. **Subscribe to events:** `message.channels` AND `message.groups` (both required)
5. Install to workspace and invite bot to your capture channel

## Phase 3: MCP Retrieval Server (Edge Function)

### 3.1 Generate an access key

```bash
ACCESS_KEY=$(openssl rand -hex 32)
echo "Your MCP access key: $ACCESS_KEY"
supabase secrets set MCP_ACCESS_KEY=$ACCESS_KEY
```

### 3.2 Create and deploy the MCP server

```bash
supabase functions new open-brain-mcp
```

**Dependencies** (`supabase/functions/open-brain-mcp/deno.json`):
```json
{
  "imports": {
    "@hono/mcp": "npm:@anthropic-ai/mcp-hono@0.1.1",
    "@modelcontextprotocol/sdk": "npm:@modelcontextprotocol/sdk@1.24.3",
    "hono": "npm:hono@4.9.2",
    "zod": "npm:zod@4.1.13",
    "@supabase/supabase-js": "npm:@supabase/supabase-js@2.47.10"
  }
}
```

**MCP Tools to implement:**

| Tool | Purpose |
|------|---------|
| `search_thoughts` | Semantic vector search — accepts query text, embeds it, calls `match_thoughts()` |
| `browse_recent` | Time-filtered retrieval (e.g., "last 7 days") |
| `stats` | Total count, tag distribution, date range |
| `capture_thought` | Direct write from any MCP client (skips Slack) |

**Authentication:** Validate `x-brain-key` header OR `?key=` URL parameter against `MCP_ACCESS_KEY` secret.

```bash
supabase functions deploy open-brain-mcp --no-verify-jwt
```

### 3.3 Connect AI clients

**Claude Code:**
```bash
claude mcp add --transport http open-brain \
  "https://[PROJECT_REF].supabase.co/functions/v1/open-brain-mcp" \
  --header "x-brain-key: [ACCESS_KEY]"
```

**Claude Desktop:**
Settings → Connectors → Add custom connector → paste URL with `?key=[ACCESS_KEY]`

**ChatGPT:**
Settings → Apps & Connectors → Create connector (requires Developer Mode)
URL: `https://[PROJECT_REF].supabase.co/functions/v1/open-brain-mcp?key=[ACCESS_KEY]`

**Cursor / VS Code:**
Use `mcp-remote` bridge or native remote MCP support with the URL above.
